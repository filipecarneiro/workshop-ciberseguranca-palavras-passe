E usas a mesma palavra-passe em mais do que um sítio?

Depois do Mentimeter, faz esta pergunta em voz alta (não no Mentimeter). Normalmente a maioria responde que sim. É o gancho perfeito para o bloco seguinte — password reuse é o problema central.

🔧 Bloco 2 — Como funcionam? 15 min

O servidor nunca guarda a tua palavra-passe em claro.

Quando crias uma conta, a tua palavra-passe é transformada antes de ser guardada.

Objetivo: desmistificar o que acontece "por baixo" quando fazemos início de sessão. Ponto chave: o servidor não sabe a tua palavra-passe, só o hash.

O que é um Hash?

Como um triturador de papel — o original não pode ser reconstruído.

Palavra-passeHash MD5
password123482c811da5d5b4bc6d497ffa98491e38
password1243f19df7c74f3f0f3b1a0e9d4dd26e91e

Muda uma letra → o hash muda completamente.

Mostra ao vivo no terminal (VM ou local): echo -n "password123" | md5sum echo -n "password124" | md5sum Sublinha o "efeito avalanche" — uma mudança mínima na entrada produz um hash radicalmente diferente.

Fugas de Dados

  • Bases de dados de serviços são roubadas por atacantes
  • O que fica exposto são os hashes, não as palavras-passe
  • Mas se o hash for fraco... pode ser revertido
Caso RockYou (2009)
32 milhões de utilizadores · palavras-passe guardadas em claro
A lista tornou-se o dicionário de ataque mais usado até hoje.
Como obtêm as bases de dados? Injeção SQL, credenciais roubadas de administradores, servidores mal configurados. Não é necessário entrar em detalhe — o importante é que acontece e é mais comum do que se pensa.

Rainbow Tables vs. Salting

Rainbow Tables

Tabelas pré-calculadas de hashes — um atacante consulta a tabela e encontra a palavra-passe em segundos.

❌ Torna hashes simples inúteis

Salt

Valor aleatório adicionado antes do hash. Mesmo a mesma palavra-passe produz hashes diferentes.

✓ Inutiliza as rainbow tables

Público técnico: podes expandir com bcrypt e o custo computacional intencional (work factor). Para manter simples: "o sal é como adicionar um ingrediente secreto diferente a cada pizza — mesmo que a receita base seja igual, o resultado é sempre diferente".

🔍 Bloco 4 — Já foste hackeado? 20 min

Vai a

haveibeenpwned.com

e verifica o teu endereço de e-mail.

Objetivo: cada aluno descobre, em tempo real, se os seus dados já foram expostos. Instrução: cada aluno acede a haveibeenpwned.com e verifica o próprio endereço de e-mail. Anuncia o tempo restante aos 15 min e aos 5 min — os alunos ficam presos a explorar.

O que procurar

  • 🟢 Verde — sem registos de fugas com esse e-mail
  • 🔴 Vermelho — o teu e-mail apareceu em pelo menos uma fuga

Para cada fuga, verifica:

  • Que serviço foi afetado?
  • Que dados foram expostos? (palavras-passe, moradas, telefones)
  • Ainda usas esse serviço? Mudaste a palavra-passe?
No separador Passwords, cada aluno verifica se alguma palavra-passe conhecida já aparece na lista. Pergunta que podem analisar: quantas fugas? que serviços? que tipo de dados expostos?

Se estiveres comprometido

  1. Revogar sessões ativas (sair em todos os dispositivos)
  2. Alterar as palavras-passe nos serviços afetados
  3. Ativar autenticação em dois passos (MFA)
  4. Alertar contactos próximos, se aplicável
Discussão em grupo (5 min no fim do bloco): "O que fazias a seguir se descobrisses que estás comprometido?" Guia para as 4 ações listadas — não deixa a discussão derivar muito.

Gestores de Palavras-passe

Bitwarden — gratuito, código aberto

  • Um cofre cifrado para todas as palavras-passe
  • Gera palavras-passe aleatórias fortes
  • Uma única palavra-passe mestra forte + MFA

É seguro confiar tudo a uma aplicação?

Sim — melhor do que reutilizar filipe2008 em 20 sítios.

Demonstra o Bitwarden: como funciona o cofre, como gera palavras-passe, porque é seguro. Debate o compromisso: um gestor com palavra-passe mestra forte e MFA é MUITO mais seguro do que reutilizar. Nota LastPass 2023: sofreu fuga grave precisamente porque a arquitetura era diferente do Bitwarden. O cofre do Bitwarden é cifrado localmente antes de ser enviado para os servidores.

Passkeys — o futuro

Sem palavra-passe. Par de chaves criptográficas:

  • Chave pública → fica no servidor
  • Chave privada → fica no teu dispositivo

Já disponível no Google, Apple e Microsoft.

Semelhante ao SSH, para quem conhece.

Para público técnico: explica o protocolo FIDO2/WebAuthn. Para público geral: "em vez de uma palavra secreta, o teu telemóvel prova que é ele — como a impressão digital". Menção breve — não é necessário aprofundar.
